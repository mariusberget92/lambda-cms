# Configuration

Lambda CMS is configured in two places: the `.env` file (low-level environment variables) and the **Settings** panel in the admin UI (runtime settings stored in the database).

## Environment variables

The `.env` file controls infrastructure-level settings. The most important variables are listed below.

### Application

| Variable | Default | Description |
|---|---|---|
| `APP_NAME` | `Lambda CMS` | Application name used in emails and the browser title |
| `APP_ENV` | `production` | Environment (`local`, `production`) |
| `APP_KEY` | — | Generated by `php artisan key:generate` — do not share |
| `APP_DEBUG` | `false` | Enable detailed error pages. Set to `false` in production |
| `APP_URL` | `http://localhost` | Full URL of your site including scheme |

### Database

| Variable | Default | Description |
|---|---|---|
| `DB_CONNECTION` | `sqlite` | `sqlite` or `mysql` |
| `DB_HOST` | `127.0.0.1` | MySQL host (MySQL only) |
| `DB_PORT` | `3306` | MySQL port (MySQL only) |
| `DB_DATABASE` | `database/database.sqlite` | Path for SQLite, or database name for MySQL |
| `DB_USERNAME` | — | MySQL username |
| `DB_PASSWORD` | — | MySQL password |

### Mail

| Variable | Default | Description |
|---|---|---|
| `MAIL_MAILER` | `log` | `smtp`, `mailgun`, or `log` (development) |
| `MAIL_HOST` | — | SMTP host |
| `MAIL_PORT` | `587` | SMTP port |
| `MAIL_USERNAME` | — | SMTP username |
| `MAIL_PASSWORD` | — | SMTP password |
| `MAIL_ENCRYPTION` | `tls` | `tls` or `ssl` |
| `MAIL_FROM_ADDRESS` | — | Sender address |
| `MAIL_FROM_NAME` | `${APP_NAME}` | Sender name |

### Queue and cache

| Variable | Default | Description |
|---|---|---|
| `QUEUE_CONNECTION` | `sync` | `sync` (inline) or `database` / `redis` for async |
| `CACHE_STORE` | `database` | Cache driver |

### Media

| Variable | Default | Description |
|---|---|---|
| `MEDIA_MAX_UPLOAD_MB` | `10` | Maximum upload size in megabytes |
| `FILESYSTEM_DISK` | `local` | File storage driver |

## Settings panel

Runtime settings are managed from **Admin → Settings** and are stored in the database. They override the corresponding `.env` defaults.

### Site

| Setting | Description |
|---|---|
| Site name | Displayed in the browser tab and emails |
| Site URL | Used for generating canonical URLs and RSS feed links |
| Timezone | Controls scheduled post times and date display |
| Date format | PHP date format string used throughout the admin UI |

### SEO

| Setting | Description |
|---|---|
| Title separator | Character between page title and site name (e.g. `·`, `—`) |
| Default meta description | Fallback description for pages without a custom one |
| Default OG image URL | Fallback Open Graph image |
| Default keywords | Fallback meta keywords |

### Mail

Same fields as the `.env` mail variables. Changes here override the environment. Includes a **Send test email** button that delivers a test message to the currently logged-in user's address.

### Media

| Setting | Description |
|---|---|
| Max upload size | Maximum file size for media uploads (MB) |
| Image resize width | Uploaded images wider than this value (px) are resized on upload |

### Comments

| Setting | Description |
|---|---|
| Comments enabled | Master toggle — disables the comment form site-wide when off |
| Comments per page | Pagination size for comment threads (5–100) |

## config/ files

For advanced configuration, edit the files in `config/`:

- `config/media.php` — Allowed MIME types, upload size, image resizing behaviour.
- `config/permission.php` — Spatie Permission guards and cache settings.
- `config/database.php` — Database connection configuration.
- `config/mail.php` — Mail driver configuration.
